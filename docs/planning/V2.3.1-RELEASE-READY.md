## Deployment Instructions

### Step 1: Create Git Tag
```bash
cd recipe-book
git tag -a v2.3.1 -m "V2.3.1: Cloud Backup UX Enhancements

- Timezone support (user settings, scheduling, display)
- Backup history UI improvements (icons, badges, stats)
- Enhanced backup preview (detailed summary)
- User-friendly error messages (translations, suggestions)

All 4 phases complete. 100% backward compatible. No breaking changes."
```

### Step 2: Push to Repository
```bash
git push origin main --tags
```

### Step 3: Deploy Backend
```bash
# Option A: Docker deployment
cd recipe-book/backend
docker build -t recipe-book-backend:2.3.1 .
docker push recipe-book-backend:2.3.1
# Update production containers

# Option B: Direct deployment
cd recipe-book/backend
npm install --production
pm2 restart recipe-book-backend
```

### Step 4: Deploy Frontend
```bash
cd recipe-book/frontend
npm install
npm run build
# Deploy dist/ to hosting service (Netlify, Vercel, S3, etc.)
```

### Step 5: Verify Deployment
```bash
# Check backend health
curl https://api.recipebook.com/health/ready

# Verify version
curl https://api.recipebook.com/health/version
# Should return: { "version": "2.3.1" }

# Test timezone endpoint
curl -X PATCH https://api.recipebook.com/api/auth/profile \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"timezone": "America/Los_Angeles"}'
```

### Step 6: Post-Deployment Monitoring
- Monitor error logs for 24 hours
- Check metrics dashboard
- Verify user timezone settings working
